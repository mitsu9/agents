# アクティビティサマリースキル

Claude Code との当日の会話履歴を収集・要約し、1日の活動サマリーを生成するスキル。
`/daily-review` の情報収集ステップから呼び出されることを想定している。単独でも `/activity-summary` で実行可能。

## 実行手順

### 1. セッションデータの収集

以下のコマンドでClaude Codeの会話履歴を収集する:

```bash
python3 skills/activity-summary/scripts/collect_claude_sessions.py --date YYYY-MM-DD
```

- `--date` を省略すると当日が対象になる
- 全プロジェクトのセッションを横断的に収集する
- 出力は JSON 形式（セッション一覧とメッセージ）

### 2. セッションの要約

収集したデータを分析し、セッションごとに以下を抽出する:

- **トピック**: 何について会話したか（1行）
- **プロジェクト**: どのリポジトリ/プロジェクトでの会話か
- **時間帯**: 開始〜終了
- **要点**: 会話で得られた結論・成果物・決定事項（箇条書き2-3個）
- **未解決**: 会話中に結論が出なかったこと、持ち越し事項があれば

### 3. 1日の活動サマリー生成

セッション要約を統合し、以下の構造でサマリーを生成する:

```markdown
## Claude Code 活動サマリー（YYYY-MM-DD）

**セッション数**: N件
**活動時間帯**: HH:MM 〜 HH:MM

### セッション一覧

1. **[トピック]**（HH:MM〜HH:MM / プロジェクト名）
	- 要点1
	- 要点2
	- （未解決）持ち越し事項

2. ...

### 今日の活動傾向
- カテゴリ別の時間配分（思考・開発・調査・計画など）
- 特筆すべき成果や進展
```

### 4. 出力

- `/daily-review` から呼ばれた場合: サマリーをユーザーに提示し、デイリーノートへの記載判断はdaily-reviewに委ねる
- 単独実行の場合: サマリーをユーザーに提示する。デイリーノートへの追記を提案する

## 重要な原則

- **テキストのみ抽出**: ツール呼び出し結果やシステムメッセージは除外し、ユーザーとアシスタントの会話テキストのみを対象にする
- **プライバシー意識**: 他プロジェクトの会話内容もサマリーに含まれるため、機密性の高い内容は要約に留める
- **軽量さ**: 詳細な議事録ではなく、振り返りに使える粒度の要約にする
- **事実ベース**: 会話内容を忠実に要約する。解釈や評価を勝手に加えない
