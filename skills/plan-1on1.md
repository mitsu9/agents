# 1on1計画スキル

指定されたメンバーとの1on1に向けて、文脈を踏まえたアジェンダを準備するスキル。

## 使い方

`/plan-1on1 [人名]` で実行する。

## 実行手順

### 1. 基盤の読み込み

- `{peoples-list}`（人名の特定）
- `{peoples}/[該当メンバー].md`（人物ファイル）
- `{sessions}/2026/2026-02-04_分析_1on1改善.md`（1on1設計の方針）

### 2. 文脈の収集

以下を並行で収集する:

**a. 前回の1on1**
- `{sessions}` から該当メンバー名を含むセッションを検索し、直近の1on1準備・振り返りを見つける
- 前回の「次回に持ち越す話題」や「宿題」があれば拾う

**b. メンバーの現在の状況**
- 人物ファイルの「現在の状況」「課題」セクション
- 人物ファイルの「ログ」セクションから最近の出来事

**c. 関連するプロジェクト状況**
- メンバーが所属するチーム・担当プロジェクトを特定する
- 該当する `{projects}` ファイルやマイルストーンがあれば進捗を確認する

**d. 最近の観察**
- 直近のデイリーノート（`Plan/daily/`）にメンバーに関する記述がないか確認する
- Notionの1on1ページがあれば（人物ファイルの関連情報に記載）、URLを提示する

### 3. 1on1分類の確認

人物ファイルの「1on1のポイント」から分類を確認する:

| 分類 | 対象 | 主軸 |
|------|------|------|
| A: 委譲促進 | リード | 判断力育成、権限委譲 |
| B1: 業務成長支援 | デザイナー | 業務を通じた成長 |
| B2: キャリア支援 | エンジニア | 長期的な方向性 |

分類が未設定の場合はユーザーに確認する。

### 4. アジェンダの提案

収集した文脈と1on1分類を踏まえ、以下の構成でアジェンダを提案する:

```
## アジェンダ案

### チェックイン（5分）
- 体調・コンディションの確認
- [状況に応じた雑談トピック]

### 前回フォローアップ（5分）
- [前回からの持ち越し事項]

### メイントピック（15-20分）
- [分類に基づくトピック + 文脈から導いた具体的な問いかけ]

### クロージング（5分）
- 今日話した中で一番大事なこと
- 次回までにお互いがやること
```

**提案のポイント**:
- 分類に応じた問いかけ例を1on1改善セッションから引用しつつ、**今の文脈に合わせて具体化**する
- 「答えを与える」のではなく「問いかける」トピック設計にする
- トピックは詰め込みすぎない（メイン1-2個に絞る）
- メンバーからの話題が出る余白を残す

### 5. ユーザーとの対話

アジェンダ案を提示した後、ユーザーに確認する:

- 「このアジェンダで進めますか？変更・追加したいことはありますか？」
- ユーザーが調整したい場合は対話で修正する
- 特に伝えたいこと、確認したいことがあれば追加する

### 6. セッションの保存

確定したアジェンダを `{sessions}/YYYY-MM-DD_準備_1on1[メンバー名].md` に保存する。

**フォーマット**:

```markdown
# 1on1準備: [メンバー名]

- 日付: YYYY-MM-DD
- 種別: 1on1準備
- 対象: [[メンバー名]]
- 分類: [A/B1/B2]

## 収集した文脈

### 前回からの持ち越し
- [あれば]

### 現在の状況
- [人物ファイル・プロジェクトから]

### 最近の観察
- [あれば]

## アジェンダ

[確定したアジェンダ]

## 1on1後メモ（実施後に記入）

### 話した内容
-

### 次回に持ち越す話題
-

### 人物ファイル更新事項
-
```

### 7. デイリーノートへの紐付け

当日のデイリーノート（`{daily}`）の `## Memo` セクションにリンクを追記する:

```markdown
- [[YYYY-MM-DD_準備_1on1メンバー名]] - 1on1アジェンダ準備
```

デイリーノートが存在しない場合はスキップする。

## 重要な原則

- **問いかけ型を意識する**: 1on1改善セッションの方針に従い、答えを与えるのではなく問いかけるトピック設計にする
- **文脈を活かす**: 一般的なトピックではなく、今のこのメンバーに必要な話題を設計する
- **詰め込まない**: トピックを絞り、メンバーの話を聞く余白を残す
- **継続性を保つ**: 前回の1on1からの流れを途切れさせない
- **実施後の記録を促す**: 保存テンプレートに「1on1後メモ」セクションを含め、実施後の振り返りにつなげる
