# 評価コメント作成スキル

評価シートの情報を元に、評価会議用コメントと本人フィードバック用コメントを生成する。

## このスキルが解決する課題

評価プロセスでは「評価会議で他の評価者に伝えるコメント」と「本人にフィードバックするコメント」の2種類が必要になる。これらは目的と読み手が異なるため、同じ情報源から書き分ける必要がある。このスキルは、評価シートに記載された目標・成果・面談記録・AIレポートから、5つのアウトプットを一貫した根拠に基づいて生成する。

## アウトプット先

`Documents/areas/ピープルマネジメント/FY{{期}}評価/{{名前}}_評価コメント.md` に1ファイルで出力する。5つのアウトプットをセクションとしてまとめる。

## 実行手順

### 1. 情報収集

以下を読み込む（存在するもののみ）:

1. **評価シート**: `Documents/areas/ピープルマネジメント/FY{{期}}評価/{{名前}}_評価シート.md`
	- 目標（コミットメント・コンピテンシー）
	- 本人コメント
	- 振り返り面談の議事録・メモ
	- 既に記入済みの評価があればそれも参照
2. **AIレポート**: `Documents/areas/ピープルマネジメント/FY{{期}}評価/{{名前}}_AIレポート.md`
	- GitHub活動サマリー等の客観データ
3. **人物情報**: `{peoples}/{{名前}}.md`（存在すれば）
	- 等級、役割、これまでの文脈

ユーザーに「どのメンバーの評価コメントを作成しますか？」と確認し、対象を特定する。

### 2. ファクト整理

コメントを書く前に、まずファクトを整理する。これがすべての根拠の土台になる。

各コミットメント項目について:
- **目標と達成基準**: 何をA/S基準として設定していたか
- **実績**: 本人コメント・面談記録から確認できる具体的成果
- **達成度の判断材料**: 基準に対してどこまで到達したか、何が足りなかったか

コンピテンシーについて:
- **期待される行動**: コンピテンシー定義に書かれた内容
- **実際の行動**: 本人コメント・面談記録・AIレポートから確認できる具体的行動
- **発揮度の判断材料**: 期待に対してどの程度発揮できていたか

### 3. アウトプット生成

以下の5つを生成する。ユーザーの評価判断を尊重しつつ、ファクトに基づいて補強・言語化する。

---

#### アウトプット①: 評価コメント_コミットメント（評価会議用）

**目的**: 他の評価者（上長・HRBP等）に評価根拠を伝える

```
## コミットメント評価

### 総合評価: {{評価ランク}}

### 各項目の評価

#### {{目標①名}}: {{評価ランク}}
- {{達成基準に対する到達状況を具体的に記述}}
- {{ファクト（数値・成果物・行動）を明示}}

#### {{目標②名}}: {{評価ランク}}
- {{同上}}

#### {{目標③名}}: {{評価ランク}}
- {{同上}}

### 総合評価の根拠
- {{各項目の重みづけを踏まえた総合判断の理由}}
- {{特筆すべき貢献や課題があれば記載}}
```

**書き方のポイント:**
- 達成基準（A基準・S基準等）と実績を対比させる。「〜を目指していたが、実際には〜」という構造が明快
- 「頑張った」「努力した」ではなく、「〜を完了した」「〜が未達」のように状態で書く
- 本人の自己評価と一次評価が異なる場合は、なぜ異なるかの根拠を明示する

---

#### アウトプット②: 評価コメント_コンピテンシー（評価会議用）

**目的**: 他の評価者に行動特性の評価根拠を伝える

```
## コンピテンシー評価

### 総合評価: {{評価ランク}}

### 各項目の評価

#### {{コンピテンシー項目①}}: {{評価ランク}}
- {{期待される行動に対して、実際にどのような行動が確認できたか}}
- {{具体的なエピソード・成果物・影響を記載}}

#### {{コンピテンシー項目②}}: {{評価ランク}}
- {{同上}}

### 総合評価の根拠
- {{コンピテンシー全体を通じた行動傾向の評価}}
```

**書き方のポイント:**
- コンピテンシーは「行動」で評価する。成果ではなく、どう振る舞ったかに焦点を当てる
- AIレポート（GitHub活動等）があれば、レビュー・情報共有・技術貢献の具体的証拠として活用する
- 等級に対する期待値を意識し、「この等級ならここまで求められる」という観点で書く

---

#### アウトプット③: 評価コメント_総評（評価会議用）

**目的**: 評価全体のまとめと今後の方針を評価会議で共有する

```
## 総評

### 今後の育成ポイント
- {{次の半期で伸ばすべきスキル・行動}}
- {{具体的な成長機会や取り組みの方向性}}

### フィードバックで伝えること
- {{本人に伝える主要メッセージ（2〜3点）}}
- {{伝え方の留意点があれば記載}}

### 処遇変更（該当する場合のみ）
- 等級変更: {{変更内容と理由}}
- ベース給変更: {{変更内容と理由}}
- コース変更: {{変更内容と理由}}
```

**書き方のポイント:**
- 育成ポイントは「弱みの克服」だけでなく「強みの伸長」も含める
- FBメッセージは、本人が納得感を持てる順序・表現を意識する（Goodから入る等）
- 処遇変更がない場合はセクションごと省略してよい

---

#### アウトプット④: FBコメント_コミットメント（本人FB用）

**目的**: 本人に直接伝えるフィードバック。成長を促す

```
## コミットメントのフィードバック

### Good
- {{具体的な成果・行動を挙げて、何が良かったか・なぜ良かったかを伝える}}
- {{本人が気づいていない貢献や価値も言語化する}}

### Motto
- {{次に目指してほしいレベル・行動を具体的に示す}}
- {{「〜ができていなかった」ではなく「次は〜を目指そう」という未来志向で書く}}
```

**書き方のポイント:**
- Goodは「事実 + なぜそれが価値あるか」のセット。単なる褒めではなく、本人の行動がチーム・事業にどう貢献したかを伝える
- Mottoは評価会議用コメントの「課題」をそのまま書くのではなく、本人が前向きに受け取れる表現にする。ただし曖昧にはしない
- 本人の自己評価コメントに言及すると「ちゃんと読んでいる」と伝わり、納得感が高まる

---

#### アウトプット⑤: FBコメント_コンピテンシー（本人FB用）

**目的**: 行動特性に対する本人向けフィードバック

```
## コンピテンシーのフィードバック

### Good
- {{発揮できていた行動とその影響を具体的に伝える}}

### Motto
- {{次に期待する行動レベルを具体的に示す}}
```

**書き方のポイント:**
- アウトプット④と同じ原則
- コンピテンシーは抽象的になりがちなので、必ず具体エピソードとセットで書く

---

### 4. ファイル出力

確認を挟まず、直接 `Documents/areas/ピープルマネジメント/FY{{期}}評価/{{名前}}_評価コメント.md` に出力する。ユーザーがファイルを見て自分で調整する前提のため、レビューステップは不要。

## 全体を通じた原則

### ファクトベース
すべてのコメントは確認可能な事実に基づく。評価シート・面談記録・AIレポートに記載のない情報を根拠にしない。「おそらく〜だろう」という推測が必要な場合は、推測であることを明示してユーザーに確認する。

### 評価会議用 vs 本人FB用の書き分け
- **評価会議用**: 客観的・簡潔・根拠重視。他の評価者が判断材料として使える情報密度
- **本人FB用**: 成長志向・具体的・前向き。本人が「次に何をすればいいか」がわかる内容

### Before/Afterで書く
成果の言語化は「やったことリスト」ではなく「状態変化」で書く。「〜を実施した」ではなく「〜の状態が、〜に変わった」という表現を心がける。
