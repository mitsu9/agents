# デイリースタートスキル

毎朝の始業時に実行し、今日の仕事を立ち上げるためのスキル。
前夜の `/daily-review` で翌日のデイリーノートは作成済み。朝は**確認・差分反映・意思決定**に集中する。

## タスク管理: Linear

タスクはLinearで管理する。Linear MCP を使ってIssueの読み取り・更新を行う。

- **チーム**: Mitsu9
- **ステータス対応**: Backlog（未整理）→ Todo（今週コミット）→ In Progress（着手中）→ Done（完了）
- **ラベル**: 期限固定 / 戦略的 / 対応型（優先分類）、思考 / 実行（種別）
- **Issue Description**: `## 完了条件` セクションに done_when を記載

## 実行手順

### 1. デイリーノートの確認

以下を並行で読み込む:

- 当日のデイリーノート（`{daily-note}`）← `/daily-review` で作成済み
- 当日のデイリーディレクトリ（`{daily}`）に前夜の収集データがあればざっと確認
デイリーノートの内容をユーザーに提示し、「これでOK？変更ある？」と確認する。

**デイリーノートが存在しない場合**（前夜に `/daily-review` を実行していないケース）:
- フォールバックとして従来どおりデイリーノートを新規作成する
- `templates/daily.md` をベースに、Linear Issue（`list_issues` で `assignee: me`, `state: Todo`）・週次ノート・マイルストーンを読み込んでフル組み立てを行う

### 2. overnight アップデートの確認

前夜の `/daily-review` 以降に変化があった項目をチェックし、デイリーノートを更新する:

**a. カレンダー差分**
- `gws calendar events list --params '{"calendarId":"primary","timeMin":"YYYY-MM-DDT00:00:00+09:00","timeMax":"YYYY-MM-DDT00:00:00+09:00（翌日）","singleEvents":true,"orderBy":"startTime"}' --format json` で当日のイベントを取得する（`+agenda` は現在時刻以降しか返さないため使わない）
- 各イベントの `attendees` 配列から `self: true` のエントリを探し、`responseStatus` を確認する
- **`declined` の予定は除外する**（不参加として扱う）
- `accepted`・`tentative`・`needsAction`・attendeesなし（自分作成の予定）のみを対象とする
- デイリーノートの「今日のプラン」と比較し、追加・変更・キャンセルされたMTGがあれば反映する
- **ランチ30分を必ず確保する**。プランにランチブロックがなければ、MTGの隙間に【ランチ】を挿入する

**b. ユーザーからの変更**
- ユーザーが追加・変更したい内容があれば反映する

差分がなければ「変更なし、このままいきましょう」で次へ。

### 3. MIT（Most Important Task）を決める

今日のタスクの中から **「今日これだけは完了させる」を1つ** ユーザーに選んでもらう。

- デイリーノートの「今日のタスク」一覧を提示し、MITを問いかける
- 選ばれたMITをデイリーノートの「今日のタスク」セクション冒頭に `> MIT: タスク名 - アクション` の形式で明記する

### 4. MTG準備のAI委任

今日のプランからMTGを走査し、**AIに準備を任せられるもの**を特定してユーザーに提案する:

- **1on1**: `/plan-1on1 [人名]` で事前アジェンダを作成
- **定例・進捗共有**: 関連情報の収集・整理
- **その他**: アジェンダのドラフト、資料の下準備

ユーザーが委任を決めたものは、その場でAIに指示を出す（バックグラウンドエージェントや `/ai-kick` を活用）。
委任するものがなければスキップ。

### 5. デイリーノート確定

最終版をデイリーノートに書き込んで完了。「いってらっしゃい」で締める。

## 重要な原則

- **速さ最優先**: 前夜の設計をベースにするため、朝は5分で終わる軽さを目指す
- **差分に集中**: ゼロから組み立てない。前夜からの変化だけを扱う
- **MITで意思決定**: タスクを並べるだけでなく、「今日の最重要」を1つ決めて宣言する
- **AI委任を先出し**: MTG準備をAIに任せることで、自分の作業時間を確保する
- **判断はユーザーに委ねる**: MITの選択もAI委任の判断も勝手にしない
- **Linear IssueはWHAT、デイリーノートはHOW**: Linear Issueは達成基準（完了条件）を定義し、デイリーノートで具体的なアクションに分解して実行する
