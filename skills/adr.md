# ADR記録スキル

会話の中で何かが決まったとき、その決定をADRとして記録する。

**このスキルがやること:** 決定の記録
**このスキルがやらないこと:** 意思決定の支援（選択肢の洗い出し、評価、ペルソナフィードバック等）

## 発動タイミング

会話の中でユーザーが決定を下したとき:
- 「じゃあXXでいこう」
- 「XXに決めた」
- 「XXの方針で進める」

## 実行手順

### 1. 会話から情報を拾う

会話の流れから以下を抽出する:

- **何を決めたか**（決定内容）
- **なぜその決定が必要だったか**（コンテキスト）
- **他に検討した選択肢**（あれば）
- **この決定による影響・トレードオフ**

### 2. 足りない情報をヒアリング

会話から拾えなかった項目だけ、簡潔に聞く。全部聞く必要はない。最低限必要なのは:

- **決定内容**（これがないと記録できない）
- **コンテキスト**（なぜ決めたか）

以下は会話から推測できなかった場合のみ聞く:
- 検討した他の選択肢
- 見直し条件・タイミング

### 3. ADRドラフトを提示

`templates/adr.md` のフォーマットでドラフトを作成し、ユーザーに確認する。

- 番号は `{adr}` 内の既存ADRの最大番号 + 1
- 会話から拾える情報は自分で埋める。ユーザーに繰り返させない
- 簡潔に書く。長文にしない

### 4. 保存

ユーザーがOKしたら `{adr}/NNNN-トピック名.md` に保存する。

## 重要な原則

- **軽く保つ**: 記録の障壁を最小化する。質問は最小限に
- **会話の流れを壊さない**: ADR記録は本題の合間にサッと済ませる
- **ユーザーの言葉を使う**: 過度に形式化・抽象化しない
