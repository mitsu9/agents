# メモスキル

思いついたことをサッと記録するための最軽量スキル。

## 特徴

- **最速**: 最小限の対話でメモを保存
- **つながる**: デイリーノートへの紐付け・関連ページのリンク
- **次が見える**: メモから汲み取ったアクション提案

## 実行手順

### 1. メモの開始

ユーザーの入力からメモの内容を把握する。

- 引数がある場合: そのままメモの内容として記録を開始
- 引数がない場合: 「何についてメモしますか？」と聞く

### 2. メモの記録（対話フェーズ）

ユーザーが話す内容をそのまま記録する:

- 箇条書きで構造化する（ただし過度な整理はしない）
- ユーザーの言葉を尊重し、勝手に言い換えない
- 「他にありますか？」と聞いて追記を促す
- ユーザーが「終わり」「以上」などと言ったら記録を終了し、整理フェーズに移る

### 3. 整理フェーズ（「終わり」の後に自動実行）

以下を一括で行い、結果をまとめて報告する。

#### 3a. メモの保存

`{sessions}` に保存する。

**ファイル名**: `YYYY-MM-DD_メモ_[トピック名].md`

**フォーマット**:

```markdown
# [トピック]

- 日付: YYYY-MM-DD
- 種別: メモ

## 内容

[メモの本文]

## 関連ページ

- [[関連ページ1]]
- [[関連ページ2]]

## 次のアクション（提案）

- [ ] [アクション項目]
```

#### 3b. 関連ページのリンク

メモの内容から関連しそうなページを探してリンクする:

- `{projects}` からプロジェクト名で関連を探す
- `{peoples}` から人物の関連を探す
- `{sessions}` から過去の関連セッションを探す
- `{goals-week}`, `{goals-month}` から関連する目標を探す

※ `{key}` は `context/sources.md` のデータキーに対応するパス

見つかったリンクはメモファイルの「関連ページ」セクションに `[[ファイル名]]` 形式で記載する。

#### 3c. 次のアクション提案

メモの内容から今後やるべきことを汲み取り、「次のアクション（提案）」セクションに記載する:

- 具体的で実行可能なアクションにする
- 深掘りが必要なら: 「`/think-analyze` で〇〇を分析する」
- 誰かに相談が必要なら: 「〇〇さんに△△を確認する」
- タスクとして積むべきなら: LinearにIssueを作成する
- 特にアクションがなければ省略可

#### 3d. デイリーノートへの紐付け

当日のデイリーノート（`{daily}`）の `## Memo` セクションにリンクを追記する:

```markdown
- [[YYYY-MM-DD_メモ_トピック名]] - 一行サマリー
```

デイリーノートが存在しない場合はスキップし、その旨を報告する。

### 4. 完了報告

以下を簡潔に報告する:

- 保存先パス
- リンクした関連ページ（あれば）
- 提案したアクション（あれば）
- デイリーノートへの紐付け結果

## 重要な原則

- **速さ優先**: メモを取ることの障壁を最小限にする
- **ユーザーの言葉を尊重**: 過度に編集・構造化しない
- **整理は「終わり」の後**: 対話中は記録に集中し、整理作業は最後にまとめて行う
- **軽量に保つ**: 憲法チェック、ペルソナフィードバック、チェックポイントは省略
