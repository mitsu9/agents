# 未返信メンション確認スキル

Slack で自分宛てのメンションのうち、まだ返信していないものを一覧表示する。

## CLIツール

`~/.ghq/github.com/mitsu9/slctl/target/debug/slctl` を使用する。

## 実行手順

### 1. メンションの取得

当日のメンションを取得する。引数で期間指定も可能（例: `/slack-unreplied 3days`）。

```bash
slctl query mentions mine --from <期間開始 ISO8601+09:00> --to <現在時刻 ISO8601+09:00>
```

- デフォルト: 当日 00:00:00+09:00 〜 現在時刻
- 引数 `Ndays`: N日前の 00:00:00+09:00 〜 現在時刻

### 2. フィルタリング

以下の条件でフィルタする:

- **public channel のみ**: `channel.kind == "public_channel"` のものだけ抽出（`mpim`, `im`, `channel`（private）は除外）
- **自分自身の投稿を除外**: `user` フィールドが自分の `user_id` のメンションは除外

自分の `user_id` は、メンション本文中の `<@USER_ID|username>` パターンや permalink のドメインから推測せず、最初のメンションの text から抽出する（`<@WFZ9C9CHM|...>` のように自分宛てメンションには必ず自分の ID が含まれる）。

### 3. スレッド確認

各メンションについて、スレッド内に自分の返信があるか確認する。

**thread_ts の決定**:
- permalink に `?thread_ts=` が含まれる場合: その値を使う（メンションがスレッド返信内にある）
- 含まれない場合: `mention.ts` を `thread_ts` として使う（メンションがトップレベルメッセージ）

```bash
slctl query channels thread --channel <channel.id> --thread-ts <thread_ts>
```

- スレッド内に自分の `user_id` の返信があれば「返信済み」
- なければ「未返信」
- エラー（チャンネルアクセス不可等）は「確認不可」として扱う

**注意**: スレッド取得は各メンションごとに実行するため、メンション数が多い場合は時間がかかる。

### 4. 結果の表示

#### 未返信メンション一覧

未返信のメンションを以下の形式で表示:

```
### 未返信メンション

1. **#チャンネル名** - @送信者名（HH:MM）
   > テキスト抜粋（80文字程度）
   [permalink](URL)

2. ...
```

#### サマリー

```
**サマリー**: 合計 N件 / 返信済み N件 / 未返信 N件 / 確認不可 N件
```

### 5. 制約事項の表示

以下を注記として表示する:

> **注意**: permalink から `thread_ts` を取得できないケースでは、スレッド返信内メンションの判定が不正確になる場合があります。
