# コミュニケーション設計フレームワーク

## 概要
意思決定の結果や方針変更、困難なフィードバックなど、「伝え方」を受け手の視点から構造的に設計するためのフレームワーク。何を伝えるかが決まった後に、どう伝えるかを考えるために使う。

## 思考の進め方

### 1. 起点の確認
- 何を伝えるのか（内容が固まっているか）
- なぜ伝える必要があるのか
- 背景にある意思決定や分析は何か

### 2. 目的の明確化
- 相手にどう思ってほしいか、どう動いてほしいか
- このコミュニケーションが成功したとき、何が変わっているか
- ゴールを1文で定義する

### 3. 受け手の理解
- 誰が受け手か（すべての受け手を洗い出す）
- 受け手の現在の理解レベル、関心事、懸念は何か
- 受け手の立場に立って考える（エンパシーマッピング）

### 4. メッセージの構築
- 結論ファーストで組み立てる（ピラミッド原則）
- 伝えること/伝えないことを意図的に設計する
- 想定される質問への回答を準備する

### 5. 伝達方法の設計
- どのチャネルで伝えるか
- いつ、誰から、どの順番で伝えるか
- フォローアップをどうするか

### 6. 検証と最終化
- 受け手の視点でメッセージを読み直す
- 成功基準を定義する
- アクションアイテムを確定する

## 確認すべき観点

### メッセージの明確性
- 結論が1文で言えるか
- 専門用語や曖昧な表現を避けているか
- 受け手の理解レベルに合った言葉を使っているか
- 「何をしてほしいか」が明確か

### 受け手の準備状態
- 受け手はこの話題についてどの程度知っているか
- 事前に共有すべき情報はあるか
- 心理的な準備が必要か（サプライズにならないか）
- 受け手が質問や意見を出せる余地があるか

### 関係性への影響
- このコミュニケーションが信頼関係に与える影響
- 過去のコミュニケーション履歴との一貫性
- 言葉にしない暗黙のメッセージは何か
- 長期的な関係性を損なわないか

### 組織政治的配慮
- 伝達順序の政治的影響（誰に先に伝えるか）
- 非公式なルートで情報が漏れるリスク
- 他部門や上位者への影響
- 既存の力学やパワーバランス

## 使える思考ツール

### ピラミッド原則
結論を最初に述べ、理由で支え、根拠で補強する3層構造。
1. **結論（1文）**: 最も伝えたいこと
2. **理由（3つ以内）**: なぜそう言えるのか
3. **根拠**: 各理由を支える事実・データ

### エンパシーマッピング
受け手の内面を4つの観点で整理する:
- **Thinks**: 考えていること（論理的側面）
- **Feels**: 感じていること（感情的側面）
- **Fears**: 恐れていること（不安・リスク認知）
- **Wants**: 望んでいること（期待・ニーズ）

### ステークホルダーマトリクス
受け手を影響力と関心度で4象限に分類し、コミュニケーション戦略を差別化する:
- **高影響力 × 高関心**: 密にコミュニケーション
- **高影響力 × 低関心**: 満足を保つ
- **低影響力 × 高関心**: 情報を提供し続ける
- **低影響力 × 低関心**: 最低限の共有

### SBIフレームワーク（フィードバック向け）
困難なフィードバックを構造化する:
- **Situation**: いつ、どこで
- **Behavior**: 何が起きたか（観察された行動）
- **Impact**: その結果どんな影響があったか

### 変化への抵抗曲線
方針変更時の受け手の心理段階を理解する:
1. **否定**: 「本当に変わるの？」
2. **抵抗**: 「なぜ変える必要があるのか」
3. **探索**: 「どうすればいいのか」
4. **受容**: 「やってみよう」

各段階に応じたコミュニケーションが必要。否定・抵抗段階では感情の受け止めと「なぜ」の説明が重要。

## 多様な視点

### 論理 vs 感情
- 論理的に正しいことと、感情的に受け入れられることは異なる
- 困難なメッセージでは感情を先に受け止める
- データだけでなく、ストーリーで伝える力

### 情報量 vs 理解
- 情報を多く伝えることと、理解を深めることは別
- 受け手が処理できる情報量には限界がある
- 本当に必要な情報だけに絞る勇気

### 一方向 vs 対話
- 伝達か、対話か。目的によって使い分ける
- 一方向の伝達は速いが、納得感は低い
- 対話は時間がかかるが、受容度が高い

### 公式 vs 非公式
- 公式な場での発信と、非公式な場での会話
- 非公式な場でのコミュニケーションが公式の場を左右することがある
- 「根回し」は日本の組織では重要な非公式チャネル

## 憲法の原則の適用

### 誠実性
- 都合の悪い情報も隠さない
- 不確実なことは不確実だと伝える
- 言行一致を保つ
- 過度な楽観や悲観で歪めない

### 多様性
- 受け手の多様な反応パターンを想定する
- 一つのメッセージがすべての受け手に同じように届くとは限らない
- 文化的・個人的な違いを考慮する
- 対立する反応も真摯に受け止める準備をする

### 構造化
- メッセージをピラミッド構造で組み立てる
- 伝える/伝えないを明示的に設計する
- 伝達シーケンスを論理的に組む
- 想定問答を体系的に準備する

### 実践性
- 受け手が「次に何をすればいいか」を分かるようにする
- 実行可能なアクションに落とし込む
- フォローアップの仕組みを設計する
- 完璧なメッセージより、タイムリーな伝達を優先する場面もある

### 継続的改善
- コミュニケーション後の反応から学ぶ
- 想定と実際のギャップを振り返る
- 伝え方のパターンを蓄積する
- 同じ失敗を繰り返さない

## チェックリスト

コミュニケーションを実行する前に確認:

- [ ] ゴールが1文で明確に定義されている
- [ ] 受け手の立場で読み直し、理解できる
- [ ] 結論が最初に来ている（ピラミッド原則）
- [ ] 伝えること/伝えないことが意図的に設計されている
- [ ] 想定問答が準備されている
- [ ] チャネルが目的に合っている
- [ ] 伝達順序が政治的にも適切である
- [ ] 感情的配慮が必要な場合、対応されている
- [ ] 「次に何をすればいいか」が明確である
- [ ] フォローアップ計画がある
- [ ] 成功基準が定義されている
- [ ] 誠実さを損なう省略や誇張がない

## ペルソナとの関連

このフレームワークでは、ペルソナを「評価者」ではなく**「受け手」として転用**する。メッセージを受け取った側の反応をシミュレーションすることで、伝え方の質を高める。

### 経営・事業視点（business-owner）の受け手反応
- 戦略的意図が正しく伝わっているか
- 組織の方向性との整合性を感じるか
- 重要なステークホルダーへの配慮は十分か
- ビジネスインパクトの説明に納得できるか

### 現場受容性視点（team-member）の受け手反応
- 第一反応として何を感じるか
- 不安や抵抗を感じる要素はないか
- 実務への影響が十分に説明されているか
- 自分たちへのリスペクトを感じるか

### 当事者共感視点（end-user）の受け手反応
- メッセージは明確に理解できるか
- 「なぜ」が腹落ちするか（納得感）
- 次に何をすればいいか分かるか
- 自分の声が聞かれていると感じるか

*伝えることは思考の延長であり、伝え方はその思考への敬意である。*
