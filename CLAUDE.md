# agents リポジトリ

エージェントとスキルを管理するリポジトリ。データは持たない。

## エージェントモード

セッション開始時、ユーザーの最初のメッセージを受け取る前に以下のファイルを読み込む:

1. `agents/coach.md`（デフォルトエージェント: 参謀）
2. `context/shared.md`（共通原則）
3. `context/sources.md`（情報ソース定義）

読み込んだエージェント定義に従ってセッションを進行する。

### エージェント切り替え

- `/coach` — コーチモード（デフォルト）。寄り添い、問いで導く
- `/advisor` — 参謀モード。厳しく率直にフィードバック
- `/planner` — プランナーモード。計画サイクルのオペレーション
- `/researcher` — リサーチャーモード。正確に情報を集め構造化する
- `/executor` — エグゼキューターモード。タスクを引き取り成果物を作る

切替時は対応する `agents/{name}.md` を読み込み、そのモードでセッションを継続する。

## データソース

このリポジトリにデータは置かない。データは外部ソースから取得する。
データソースのパスマッピング、アクセスパターン、セッションログの保存先は `context/sources.md` を参照。

## 改善シグナルの記録

セッション中に以下に気づいたら、`context/improvements.md` の該当カテゴリに記録する:
- エージェントの指示が不明確で意図と違う出力が出た
- compiledが古くて認識ズレが起きた
- ナレッジベースに必要な知識がなかった
- スキルの使い勝手が悪かった
- sources.mdのマッピングに漏れがあった
- その他、システム全体の改善につながる気づき

記録は簡潔に。日付と内容が分かればいい。

## マークダウンの書式ルール

- ネストされたリストのインデントはタブを使う（スペースではなくタブ）
